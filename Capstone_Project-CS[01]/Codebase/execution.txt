CV CREATION USING LLM - EXECUTION GUIDE
========================================

PREREQUISITES:
--------------
1. Python 3.8 or higher installed
2. OpenAI API key obtained
3. All dependencies installed

SETUP INSTRUCTIONS:
-------------------

Step 1: Install Dependencies
$ pip install -r requirements.txt

Step 2: Configure Environment
$ cp .env.example .env
$ nano .env  # Or use any text editor
# Add your OpenAI API key to the OPENAI_API_KEY field

Step 3: Verify Installation
$ python -c "import langchain, openai, pdfplumber, docx; print('All dependencies installed!')"


EXECUTION METHODS:
------------------

METHOD 1: Complete Interactive Workflow (Recommended)
------------------------------------------------------
$ python main.py

This launches the interactive menu with options:
- Option 1: Complete workflow (Extract → Parse → Generate → Revise → Export)
- Option 2: Quick match (Check profile-job fit)
- Options 3-7: Individual features
- Option 0: Exit

Example session:
$ python main.py
Select option (0-7): 1
Enter path to your resume: input/profiles/my_resume.pdf
Select (1/2): 1
Enter path to job description file: input/jobs/job_description.txt
Continue anyway? (y/n): y
Would you like to review and revise the resume? (y/n): y
Select option (1/2/3): 1
[Enter your edits...]
DONE
Select (1/2/3): 2
Select (1/2/3): 1


METHOD 2: Run Individual Features
----------------------------------

Feature 1 - Extract Resume Data:
$ python extractor.py
Enter the path to your resume file: input/profiles/resume.pdf

Feature 2 - Parse Job Description:
$ python parser.py
Select option (1/2): 1
Enter the path to the job description file: input/jobs/job.txt

Feature 3 - Generate Tailored Resume:
$ python generator.py
Enter path to extracted profile JSON: output/resume_extracted.json
Enter path to parsed job JSON: output/job_parsed.json

Feature 4 - Revise with ATS Feedback:
$ python reviser.py
Enter path to resume JSON: output/resume.json
Enter path to job JSON: output/job_parsed.json

ATS Checker (Standalone):
$ python ats_checker.py
Enter path to resume JSON: output/resume.json
Enter path to job JSON: output/job_parsed.json

Profile-Job Matcher:
$ python matcher.py
Enter path to profile JSON: output/resume_extracted.json
Enter path to job JSON: output/job_parsed.json

Export to PDF/DOCX:
$ python formatters.py
Enter path to resume JSON: output/resume_final.json
Select (1/2/3): 3

TROUBLESHOOTING:
----------------

Error: "OpenAI API key not configured"
Solution: Ensure .env file exists and contains valid API key

Error: "ModuleNotFoundError"
Solution: Run "pip install -r requirements.txt"

Error: "File not found"
Solution: Use absolute or correct relative paths to input files

Error: "No text could be extracted"
Solution: Ensure PDF is not scanned image, try converting to TXT first

Error: "Rate limit exceeded"
Solution: Wait a few seconds and try again, or check API quota


NOTES:
------

1. All output files are saved in the output/ folder
2. Input files can be placed in input/profiles/ and input/jobs/ for organization
3. The system uses GPT-4o-mini for extraction/parsing (cost-effective)
4. The system uses GPT-4o for generation/revision (higher quality)
5. Each complete workflow costs approximately $0.02-0.05 in API credits
6. ATS scores above 75 are considered good for most systems
7. Multiple revisions can improve ATS scores significantly
8. Always export final resume in both PDF and DOCX formats
9. Revision history is tracked automatically
10. All intermediate files are kept for reference
